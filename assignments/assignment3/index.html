Concurrent Ping-Pong Program (Assignment 3)

Overview of the Assignment

In this assignment, you will implement a program that uses a pattern-oriented ping-pong framework to spawn threads that correctly alternate printing "Ping" and "Pong", respectively on either the command-line display (of the Eclipse IDE) or in an Android TextView (in either the AVD emulator or on an actual Android device).

Program Description

In this directory you'll find Java source code, AndroidManifest.xml, and associated XML metadata files. The directory A3-Android/src/edu/vuum/mocca/ contains the skeleton Java code that you'll implement by completing the "TODO - You fill in here" comments to provide a working solution. DO NOT CHANGE THE OVERALL STRUCTURE OF THE SKELETON - just fill in the "TODO - You fill in here" portions!!!

Although there are a lot of files, the actual amount of code you need to write is relatively localized - the main goal is to understand the pattern-oriented ping-pong framework and how to apply various Java synchronization mechanisms. In particular, you'll need to do the following:

  • Read and understand the PingPongThreadSema and PingPongThreadCond classes, which are covered in the Java Synchronization and Scheduling Example (Part 1) and Java Synchronization and Scheduling Example (Part 2) videos.

  • To understand these classes you'll also want to understand the Java ReentrantLock and ConditionObject, which are covered in the Java ReentrantLock and Java ConditionObject videos.

  • Finish the implementations of PingPongThreadQueue and PingPongThreadBall, which involves understanding BlockingQueues as well as implementing the PingPongPaddle class which uses ReentrantLocks and Condition objects.

  • Implement the hook methods in the ConsolePlatformStrategy.java and AndroidPlatformStrategy.java classes that use a Java CountDownLatch, which is covered in the Java CountDownLatch video.

To Run the Android Application, right click on the 'A3-Android' project and select [Run As] -> [Android Application]

To Run the Android Unit Test, right click on the 'A3-Android-Test' project and select [Run As] -> [Android Unit Test]

To Run the Java Console application, right click on the 'A3-Android' project and select [Run As] -> [Java Application]

Note: Some versions of the ADT Plugin may not be able to run on the console by default. The current workaround is as follows:

1. Right Click on the project -> "Run As" -> "Run Configurations"
2. Select "Java Application" and then "Main"
3. Open the tab "Classpath" and remove the "Android Libraries" under "Bootstrap Entities"
4. Click "Run" and your project should run properly (assuming you've implemented everything correctly).

Finally, while you have implemented different synchronization strategies for the PingPongClass, only one strategy is used in the application at any given time. To use/test different strategies, you must edit Options.java, which contains a member called "mSyncMechanism". Changing this will make the application use different strategies. For example setting it to "QUEUE" will use your BlockingQueue strategy, while setting it to "PADDLE" will use your paddle/ball strategy.

Concluding Remarks

This assignment is designed to help you get familiar with writing multi-threaded programs using Eclipse ADT, Java Threads, and synchronization mechanisms. It doesn't require detailed knowledge of Android concurrency frameworks, which we'll cover in upcoming assignments.
